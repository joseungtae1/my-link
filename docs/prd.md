# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트 명**: 마이링크 (MyLink)
- **목적**: 닉네임 기반의 고유 접속 주소 하나로 자신만의 주요 링크들을 매우 심플하게 모아서 공유할 수 있는 통합 링크 서비스.
- **주요 기술 스택**: 
  - 프론트엔드: Next.js (App Router), shadcn/ui 기반 단일 디자인 (테마 없음)
  - 백엔드: Firebase (구글 소셜 로그인 전용 및 Firestore 데이터베이스)

## 2. 필수 요건 및 핵심 기능 목록 (MVP)

1. **회원 인증 및 닉네임 기반 접속**
   - 구글 로그인 전용 (Firebase Auth). 가입 시 접속 주소용 슬러그 `닉네임(displayName)` 선점. (핸들 `@` 없음)
2. **결과물 공유 (Web Share API) 및 단일 랜딩 페이지**
   - `domain.com/닉네임` 포맷의 군더더기 없는 심플한 방문자용 페이지 제공.
   - 랜딩 하단에 **[공유하기] 버튼**을 두어 모바일에서 모바일 OS 네이티브 메신저 앱으로 즉시 프로필 링크를 보낼 수 있도록 지원.
3. **프로필 인라인 편집 (이미지 생략)**
   - 닉네임(username)과 한줄 소개(bio)를 이미지 없이 랜딩 상단에 위치. 대시보드 화면 클릭을 통한 인라인 편집(Inline Edit) 고정 지원.
4. **링크 관리 및 클릭 누적 통계 (Click Count)**
   - 대시보드에서 링크를 인라인 형태로 추가, 수정, 삭제.  
   - 외부 방문자가 링크 탭 시, 해당 링크의 **단순 클릭 수(`clickCount`)**를 카운트하여 대시보드 컴포넌트 우측 끝에 집계된 숫자 표출.
5. **구글 API 파비콘 수신 및 폴백 (Fallback) 디자인**
   - 링크 생성 시 Google S2 Favicon API를 통해 도메인의 파비콘 추출.
   - 파비콘 통신 실패 시, 엑스박스 등 에러를 노출하지 않고 기본 **[링크(🔗) 모양 아이콘]**으로 롤백 대체 적용.

## 3. 요건 상세 요약
- **단일 UI 및 인라인 디자인**: 복잡한 설정 다단계 모달 창을 없애고 텍스트를 즉석에서 고치는 극도의 심플함을 지향합니다.
- **서브 컬렉션 DB 모델**: Firestore 상의 유저 문서(`displayName` 등 포함) 하위에 개별 `links` 등 세부 컬렉션을 저장해 `clickCount`를 안정적으로 실시간 격리 추적합니다.
