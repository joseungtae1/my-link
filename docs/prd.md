# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트 명**: 마이링크 (MyLink)
- **목적**: 닉네임 기반의 고유 접속 주소 하나로 자신만의 주요 링크들을 매우 심플하게 모아서 공유할 수 있는 통합 링크 서비스.
- **주요 기술 스택**: 
  - 프론트엔드: Next.js (App Router), shadcn/ui 기반 단일 디자인 (테마 없음)
  - 백엔드: Firebase (구글 소셜 로그인 전용 및 Firestore 데이터베이스)

## 2. 필수 요건 및 핵심 기능 목록 (MVP)

1. **회원 인증 및 닉네임 기반 접속**
   - 구글 소셜 로그인 전용 (Firebase Auth)
   - 가입 시 접속 주소로 사용될 '닉네임' 선점 (예: `domain.com/닉네임`) - 핸들(`@`) 기호 없음.
2. **프로필 인라인 편집 (이미지 생략)**
   - 프로필 이미지를 사용하지 않으며, 닉네임과 한줄 소개(bio) 텍스트만 랜딩 페이지 상단에 노출.
   - 대시보드 화면 내에서 텍스트를 클릭하여 즉시 텍스트를 수정하는 '인라인 편집(Inline Edit)' 기능 필수 지원.
3. **링크 관리 및 인라인 편집**
   - 링크의 타이틀과 URL 역시 대시보드에서 인라인 형태로 추가, 수정할 수 있으며 삭제도 함께 제공됨.
   - 복잡한 관리 요소(순서 변경 드래그, 활성화 토글) 배제.
4. **구글 API 기반 파비콘 자동 연동**
   - 링크 생성 시 입력받은 URL을 바탕으로 Google S2 Favicon API(`https://www.google.com/s2/favicons`)를 연동하여 해당 도메인의 파비콘을 링크 아이템 아이콘으로 표출.
5. **사용자 고유 랜딩 페이지**
   - `domain.com/닉네임` 포맷의 군더더기 없는 심플한 단일 디자인 템플릿 제공.

## 3. 요건 상세 요약
- **단일 테마 및 인라인 UI**: 복잡한 설정 페이지나 다단계 모달(Modal) 폼을 없애고 화면의 글자를 즉석에서 클릭해 편집하는 극도의 심플함을 지향합니다.
- **서브 컬렉션 DB 모델**: Firestore 상의 유저 문서(`nickname` 등 포함) 하위에 `links` 서브 컬렉션으로 개별 링크를 분리 보관하여 데이터의 무결성과 쿼리 효율성을 높입니다.
