# 마이링크 (MyLink) - 사용자 시나리오

해당 시나리오는 방문자와 소유자가 서비스 내에서 수행할 수 있는 "모든 가능한 작업"의 목적과 행동을 명확히 파악하기 위해 "사용자는 [목적]하기 위해 [행동]한다" 형식으로 꼼꼼히 설계되었습니다.

## 1. 방문자: 마이링크 랜딩 페이지 방문 및 상호작용 과정

### 1-1. 페이지 접속 및 정보 확인
- 사용자는 크리에이터가 운영하는 주요 채널과 발자취를 한눈에 확인하기 위해 **소셜 미디어나 브라우저를 통해 마이링크 주소(`domain.com/displayName`)를 클릭하여 접속**한다.
- 사용자는 방문한 곳이 대상 크리에이터의 페이지가 맞는지 식별하기 위해 **페이지 최상단에 노출된 닉네임(username)과 짧은 소개글(bio)을 읽는다**.

### 1-2. 개별 링크 탐색 및 조회
- 사용자는 자신이 원하는 외부 플랫폼(유튜브, 쇼핑몰 등)이 목록에 있는지 빠르게 인지하기 위해 **각 버튼 좌측에 표시된 구글 연동 파비콘(아이콘)과 우측의 텍스트 타이틀을 시각적으로 훑어본다**.
- 사용자는 API가 특정 웹사이트의 아이콘을 찾지 못해 이미지가 누락되더라도 시각적 통일감을 느끼기 위해 **시스템이 안전하게 대체 제공하는 '기본 링크 아이콘(🔗)'을 거부감 없이 인식**한다.

### 1-3. 링크 탭(이동) 및 데이터 전송
- 사용자는 목적지 웹사이트의 실질적인 영상이나 상품 정보를 확인하기 위해 **가장 관심 있는 링크 버튼 영역을 화면에서 클릭(또는 터치)하여 브라우저의 새 창으로 곧장 이동**한다.
- 사용자는 (소유자의 대시보드 화면에 해당 링크가 얼마나 인기 있는지 집계되도록 돕기 위해) **링크 버튼을 누르는 순간 시스템 내부적으로 해당 링크의 단순 클릭 수(`clickCount`) 값을 +1 증가시키는 트래킹 통신 처리에 자동 기여**한다.

### 1-4. 프로필 타인에게 공유하기
- 사용자는 발견한 유용한 크리에이터의 링크 프로필을 주변 지인들에게 널리 알리기 위해 **모바일 기기 화면 하단에 위치한 [공유하기] 버튼을 직접 누른 다**.
- 사용자는 주소를 일일이 복사/붙여넣기 하는 번거로움을 피하기 위해 **브라우저에 내장된 네이티브 공유 모달(Web Share API)을 띄워 카카오톡 채팅방이나 문자로 프로필 링크를 쉽게 전파**한다.

---

## 2. 소유자: 프로필 개설부터 로그아웃까지 전체 관리 과정

### 2-1. 관리자 접속 및 계정 개설
- 사용자는 자신만의 고유한 통합 링크 페이지 공간을 확보하기 위해 **마이링크 앱 메인에 진입해 [Google 계정으로 시작하기] 버튼을 누른 다**.
- 사용자는 복잡한 서식이나 비밀번호 입력 없이 매우 간편하게 회원가입을 마치기 위해 **구글 소셜 로그인을 완료하고, 내 이메일 앞부분(Gmail prefix)을 바탕으로 자동 부여된 나만의 접속 주소 아이디(displayName)를 획득**한다.

### 2-2. 대시보드 진입 및 프로필 인라인 편집
- 사용자는 처음 방문자들에게 보여질 자신의 활동명과 핵심 인사말 카피라이팅을 최신화하기 위해 **대시보드 상단 영역에 위치한 이름(username) 혹은 자기소개(bio) 텍스트를 마우스로 직접 클릭**한다.
- 사용자는 페이지 새로고침이나 팝업창 없이 극도로 직관적인 동작 속에 정보를 수정하기 위해 **텍스트가 바로 바뀐 인라인 폼(에디터)에서 내용을 치고 엔터(Enter)를 눌러 즉각적으로 데이터베이스에 반영**한다.

### 2-3. 신규 링크 추가 및 파비콘 연동 확인
- 사용자는 새로 오픈한 쇼핑몰이나 소셜 미디어 주소를 퍼블릭 페이지에 당장 노출하기 위해 **대시보드 하단 영역의 [+ 새 링크 추가] 빈 줄 텍스트를 클릭**한다.
- 사용자는 마우스 포인터를 이동시킬 필요 없이 한 자리에서 작업을 끝내기 위해 **추가된 인라인 입력칸에 링크 타이틀과 목적지 URL 주소를 키보드로 타이핑하고 엔터를 눌러 최종 등록**한다.
- 사용자는 링크 옆에 넣을 적합한 이미지를 일일이 다운로드받는 수고를 면하기 위해 **저장과 동시에 구글 API가 URL의 호스트를 분석하여 해당 사이트의 공식 파비콘 썸네일을 좌측에 자동으로 붙여주는 시각적 피드백을 수용**한다.

### 2-4. 기 등록 링크 수정 (이름/URL 갱신)
- 사용자는 예전에 올린 링크 주소가 만료되었거나 오타를 고치기 위해 **대시보드의 기존 링크 항목 안에 있는 타이틀이나 URL 쪽 텍스트를 다시 한 번 클릭**한다.
- 사용자는 시스템에 새로운 목적지를 안전하게 덮어쓰기 위해 **즉석에서 활성화된 인라인 에디터 내에서 문구를 새 내용으로 갱신하고 외부 영역을 클릭(blur)하여 반영 절차를 마무리**한다.

### 2-5. 퍼포먼스(클릭 수) 기반 통계 확인
- 사용자는 자신이 등록한 여러 외부 채널 중 어떤 채널의 유입이나 방문율이 가장 활발한지 모니터링하기 위해 **대시보드 링크 목록의 각 링크 항목 우측 끝에 조그맣게 노출된 '클릭 수 카운트(clickCount)' 숫자를 주기적으로 읽는다**.

### 2-6. 링크 영구 삭제
- 사용자는 더 이상 사업하지 않는 비즈니스 링크나 비공개하고 싶은 콘텐츠 링크를 방문자에게 보이지 않게 제거하기 위해 **삭제할 해당 링크 영역 우측에 나타난 휴지통 모양의 [삭제] 아이콘을 표적 삼아 누른 다**.
- 사용자는 이 버튼을 실수로 눌러 중요한 홍보 URL 데이터가 한 방에 지워지는 치명적 실수를 방지하기 위해 **화면 중앙에 스위칭 된 브라우저 기본 확인(Confirm) 대화창에서 '확인' 버튼을 최종적으로 눌러 의도적인 폐기를 단행**한다.

### 2-7. 테스트 및 퍼블릭 뷰 검증
- 사용자는 지금까지 설정한 모든 세팅(프로필 인라인 텍스트, 링크의 URL 및 파비콘, 공유 버튼 등)이 실제 모바일 환경 일반인에게 어떻게 보이는지 정확히 검증하기 위해 **대시보드 상단 혹은 좌측에 노출된 [내 마이링크 방문하기] 라우팅 텍스트 버튼을 클릭**한다.
- 사용자는 본인이 입력한 URL이 실제 올바른 목적지로 떨어지는지 파악하기 위해 **열린 자신의 퍼블릭 페이지에서 링크 버튼들을 탭하여 스스로 이동(클릭) 작동 테스트를 수행**한다.

### 2-8. 시스템 보호 및 로그아웃
- 사용자는 공용 PC나 개방된 네트워크 환경에서 자신의 파이어베이스 인증 관리자 세션을 타인에게 빼앗기지 않고 안전하게 보호하기 위해 **모든 링크 관리 작업을 성실히 마친 뒤 대시보드 핵심 GNB에 위치한 [로그아웃] 아이콘 텍스트를 찾아 눌러 소유자 권한을 완전히 내려놓는다**.
